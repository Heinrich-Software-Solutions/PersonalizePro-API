Using the Token System

The PersonalizePro API provides a token system to secure all API routes from unauthorized access. This guide will explain how to enable and use the token system to ensure that only authorized users can access the API.

Enabling the Token System​

To enable the token system, you need to modify the settings in the settings.js file located in the /api/src/configuration/ directory.

  1. Navigate to the Configuration Directory: Open the /api/src/configuration/ directory in your project.

  2. Open the settings.js File: In the settings.js file, you will find an object that defines various configurations for the API.

  3. Enable Authentication: Locate the auth property within the api object and set it to true to enable authentication.

module.exports = {
  api: {
    port: 5000,
    auth: true, // TURN ON AUTH HERE
  },
  database: {
    port: 27017,
    host: "mongo",
  },
};

Defining API Tokens​

The API tokens are defined in the api-tokens.js file, also located in the /api/src/configuration/ directory. This file contains an array of objects, each representing an API token.

  1. Open the api-tokens.js File: In the api-tokens.js file, you will find an array containing objects representing API tokens.

  2. Add New Tokens: To add a new API token, simply add a new object to the array with the token property set to the desired token value.

module.exports = [
  {
    token: "1234567890",
  },
  {
    token: "0987654321",
  },
  // Add more tokens if needed
];

You can add as many tokens as necessary to accommodate all authorized users or applications.

Accessing Requester Information​

When a request is made to an authenticated route, the token used to make the request is available in the req.auth object. You can access the requester information as follows:

api.get("/token-check", (req, res) => {
  const { token } = req.auth;
  console.log(`REQUEST WAS MADE BY: ${token}`);
});

The req.auth object contains the token property, which holds the API token used to make the request. This allows you to track which token was used for each request, helpful for monitoring and debugging purposes.

By enabling and using the token system provided by the PersonalizePro API, you can secure all API routes from unauthorized access. Only users or applications with valid API tokens defined in the api-tokens.js file will be allowed to access the protected routes. This adds an extra layer of security to your API, ensuring that sensitive data and functionalities are accessible only to authorized entities. Always keep your API tokens secure and only share them with trusted parties.
