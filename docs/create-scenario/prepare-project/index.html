Prepare The New Project

We will prepare our created projet for the API

Step 1: Edit the MLT XML File

Open the MLT XML file in a code editor. In the XML file, you will see a root element at the top. Set our first placeholder {{basePath}} inside the root element. This dynamic basepath is used by the system to determine the location of the project when it gets rendered.

In the XML file, you can use variables for the files you uploaded to the API (images, audio). To use these variables, you need to define a prefix for each type:

  • Audio: audio_
  • Images: image_

For example, if you have uploaded an file with the variable name "name" and an image with the variable name "image_1," you can use them inside the XML as {{audio_name}} and {{image_1}}, respectively.

Step 2: Register the Project to the API

To add your project to the Render Video API, you need to go to the following file:

Path: /api/src/data/projects/index.projects.js

In this file, you will find an array with example projects. To add your project, add a new object to the array like this:

{
  id: "12345", // GIVE THE PROJECT AN UNIQUE ID
  lang: "en", // SET THE LANGUAGE
  path: `${__dirname}/PROJECT-1/PROJECT.mlt`, // SET THE PROJECT PATH
  type: "videofilm", // SET THE PROJECT TYPE
  validation: {
    // USE UPLOADED FILES INSIDE THE MEDIA
    image_1: "string",
    audio_name: "string",
    // Add more variables if needed
  },
  meta: {
    // USE FOR YOUR INTERNAL REFERENCING
    msg: "example project",
  },
}

Replace the placeholders with the appropriate values:

  • id: Give the project a unique ID.
  • lang: Set the language of the project.
  • path: Set the path to your MLT XML file.
  • type: Set the type of the project (e.g., videofilm, videocall, phonecall).
  • validation: Define the variables used in the XML and their data types (e.g., "string" for text).
  • meta: You can use this object for your internal referencing and provide additional information about the project.

Step 3: Copy the Prepared Folder

Copy the folder with the MLT XML file and all the assets used in the project to the following location:

Path: /api/src/data/projects

Step 4: Test Your New Scenario

With the project folder and the project information added to the API, you can now test your new scenario. The API will recognize the variables and use the uploaded files to render the video accordingly.

Congratulations! You have successfully created a new scenario for the Render Video API using Kdenlive and integrated it into the API for dynamic video rendering. Now you can enjoy the flexibility of generating personalized videos based on the uploaded files and the scenario template you created.
